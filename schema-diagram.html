⚠ Gaps Identified (Separate Tasks)
LocationEmployerEnablement (MISSING) — Employer (Category type=employer) has no location binding. A junction collection linking employer → location is needed, identical to LocationRoleEnablement. Without this, employers remain global to the org instead of being location-scoped. This is a separate task.

IntegrationConfig (PROPOSED, not yet built) — Xero/MYOB credentials and entity mappings per location or org. OAuthToken handles the token, but a config document storing the mapping (Employee → Xero Employee ID, Award → Xero Pay Item) is needed for the export workflow.

DailyShift.locationId / roleId stored as String — Currently these will be stored as Category names (strings) to match how User.location[] and Employee.location[] work. This is intentional for now. A future migration to ObjectId references with proper $lookup queries is recommended.
